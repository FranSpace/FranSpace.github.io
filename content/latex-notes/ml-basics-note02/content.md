## Summary

这篇笔记围绕训练循环中的三个基本单位展开：epoch、batch 和 iteration。它们看起来像训练日志里的术语，但其实对应着模型更新的真实节奏，是理解后续损失下降、学习率调整和训练稳定性的基础。

## 为什么训练不是一次完成的

神经网络里有大量可训练参数，通常不可能靠一次计算就到达最优值。训练更像一个不断试错和修正的过程：

- 先用当前参数做前向传播。
- 再计算当前误差。
- 然后通过反向传播得到梯度。
- 最后由优化器更新一次参数。

## epoch、batch 和 iteration 的定义

- epoch：完整使用一遍训练集。
- batch：一次送入模型的一小批数据。
- iteration：处理一个 batch 并完成一次参数更新。

如果训练集大小记为 $N$，batch size 记为 $B$，那么每个 epoch 中大约会有

$$
\text{iterations per epoch} = \frac{N}{B}.
$$

## batch size 为什么重要

- batch 太小，更新更频繁，但梯度噪声更大。
- batch 太大，梯度更平滑，但显存和计算成本更高。
- batch size 不只影响速度，也会影响泛化和训练稳定性。

## PINN 中的特殊之处

PINN 里的 batch 往往不像普通监督学习那样只是“样本切块”。很多时候，一个 batch 里同时包含：

- 内部残差点
- 边界点
- 初始条件点
- 可能的观测数据点

所以在 PINN 中，batch 不只是数据量控制单位，也是在控制不同物理约束的采样构成。

## 一页总结

- epoch 是“看完一遍训练集”。
- batch 是“每次喂给模型的一小批样本”。
- iteration 是“处理一个 batch 并更新一次参数”。
- 在 PINN 中，batch 还承担了组织不同物理约束采样点的作用。
