## Summary

学习率（Learning Rate, LR）是神经网络训练中最关键的超参数之一。它不改变梯度方向，但决定沿这个方向每次走多远。学习率过小会让训练非常慢，过大又会造成震荡甚至发散。对 PINN 来说，这个问题更敏感，因为总损失往往由 PDE 残差、边界条件和数据项共同组成，学习率会直接影响这些部分之间的训练平衡。

## 学习率在参数更新中的位置

在梯度下降中，参数更新写成

$$
\theta_{k+1}
=
\theta_k
-
\eta \nabla_\theta L(\theta_k).
$$

其中：

- $\theta_k$：当前参数。
- $\nabla_\theta L(\theta_k)$：当前位置的梯度。
- $\eta$：学习率。

梯度决定往哪个方向走，而学习率决定这一步到底走多远。可以把梯度理解为方向盘，把学习率理解为油门。

## 从一维函数理解学习率

对于一维函数 $L(\theta)$，更新写成

$$
\theta_{k+1}
=
\theta_k
-
\eta \frac{\mathrm{d}L}{\mathrm{d}\theta}.
$$

如果学习率很小，即便梯度很大，参数每次也只移动一点点；如果学习率很大，参数就可能一次走很远。问题在于损失函数通常不是直线，而是弯曲的曲面，所以步子过大时完全可能跨过低谷，直接跳到更差的位置。

## 学习率过小会发生什么

学习率过小时，训练最直接的表现就是慢。每次更新幅度都很小，即使方向是对的，也需要很多次迭代才能看到明显下降。

典型现象包括：

- loss 下降很慢。
- 训练很久以后仍然没有明显改善。
- 参数更新幅度极小。
- 误差云图或者预测结果变化不明显。
- 在有限训练时间内看起来像“没学会”。

这并不一定意味着模型结构有问题，很多时候只是步长太保守。

## 学习率过大会发生什么

学习率过大时，参数每一步更新都太猛。即使负梯度方向在当前点是下降方向，也可能因为步长太长而越过低损失区域，从而引起震荡甚至发散。

常见现象包括：

- loss 剧烈上下波动。
- loss 偶尔下降，但很快又被拉高。
- 出现 `NaN` 或 `Inf`。
- 参数值异常增大。
- 在 PINN 中，应力场或 PDE 残差出现明显异常尖峰。

因此，loss 发散并不总是因为模型公式错了，也很可能只是学习率过大。

## 学习率和 loss 曲线的关系

观察 loss 曲线是判断学习率是否合适的重要手段：

- loss 平稳下降：学习率大概率比较合适。
- loss 几乎不变：学习率可能太小，也可能梯度接近零。
- loss 强烈震荡：学习率通常偏大。
- loss 先下降后发散：训练后期学习率可能过大。
- 总 loss 下降但某些子 loss 不动：在 PINN 中往往意味着损失项权重或采样存在问题。

对 PINN 来说，必须同时看总 loss 和各个子 loss，而不能只看一个总指标。

## 固定学习率

最简单的策略是固定学习率，即整个训练过程中始终使用同一个值：

$$
\eta_k = \eta_0.
$$

它的优点是简单、可复现、容易理解；缺点是很难同时兼顾训练前期和后期。训练早期通常需要较大步长快速下降，而训练后期更需要较小步长来精细收敛。

## 学习率调度

学习率调度（Learning Rate Schedule）指的是在训练过程中动态调整学习率，而不是始终使用固定值。常见策略包括：

- 阶梯衰减：每隔一定 epoch 把学习率乘一个衰减系数。
- 指数衰减：学习率按指数规律逐渐减小。
- 余弦退火：用平滑余弦曲线让学习率逐步下降。
- `ReduceLROnPlateau`：当 loss 长时间不再改善时自动降低学习率。
- warmup：训练初期先从较小学习率逐步升高。

这些策略的共同逻辑是：训练不同阶段需要不同步长。前期更重探索，后期更重精细收敛。

## warmup 的作用

warmup 的意思是：训练开始时不立刻使用目标学习率，而是从更小的值逐渐升高到目标值。

它适用于训练初期梯度不稳定、网络较深或损失项较复杂的情况。对大型网络、复杂 PINN 或多项损失组合而言，warmup 可以降低刚开始就把参数推到坏区域的风险。

## 学习率与 batch size 的关系

batch size 会影响梯度噪声，因此学习率往往需要和它一起考虑。

- 小 batch：梯度噪声大，学习率过大时更容易震荡。
- 大 batch：梯度更稳定，可以尝试更大的学习率。
- full-batch：梯度最稳定，但每次更新代价高。

在 PINN 中，batch size 的含义还会更复杂，因为训练点往往包含内部残差点、边界点、初值点和数据点，不同点集的数量和比例都会影响梯度方向。

## 学习率与优化器的关系

不同优化器对学习率的敏感程度不同。

普通梯度下降直接使用

$$
\theta_{k+1}
=
\theta_k
-
\eta \nabla_\theta L,
$$

因此对学习率非常敏感。

Adam 会利用梯度的一阶矩和二阶矩来做自适应缩放，所以通常比普通 GD 或 SGD 更容易训练深层网络。但 Adam 仍然依赖学习率，它只是把这个学习率变成更“全局尺度”的控制量。

L-BFGS 又不同。它属于拟牛顿法，会利用历史信息近似曲率，并常配合线搜索，因此它虽然也有步长控制思想，但与 Adam 中的显式学习率不是完全同一个概念。

## PINN 中学习率为什么更敏感

PINN 中学习率通常比普通监督学习更敏感，原因包括：

- 总损失由多个物理约束项组成，尺度可能差异很大。
- PDE 残差可能涉及高阶导数，对参数变化更敏感。
- 应力、能量等物理量常通过自动微分获得，数值扰动更容易被放大。
- 边界条件与内部残差之间可能存在训练竞争。
- 采样点分布会改变梯度估计，从而改变最合适的步长。

所以在 PINN 中，“学习率合适”不只意味着总 loss 在下降，还意味着不同约束项之间的训练平衡也在改善。

## 如何为 PINN 选择学习率

对入门问题，可以先采用保守而可复用的策略：

- Adam 阶段先尝试 $10^{-3}$。
- 如果 loss 振荡严重，就降到 $10^{-4}$。
- 如果 loss 长时间几乎不变，再考虑适度增大。
- 始终分别记录总 loss 和各个子 loss。
- 后期可逐渐减小学习率，或切换到 L-BFGS 做精细收敛。

对于固体力学 PINN，还要特别关注 Neumann 边界、应力误差和残差分布，因为这些量对学习率的敏感性通常更高。

## 学习率调试流程

实际调试时，可以按下面的顺序检查：

1. 先确认 PDE、边界条件和自动微分实现没有明显错误。
2. 用一个中等学习率开始，例如 Adam 的 $10^{-3}$。
3. 观察总 loss 和各子 loss 的变化趋势。
4. 如果 loss 发散或震荡，先减小学习率。
5. 如果 loss 长时间不动，检查梯度是否过小或学习率是否过小。
6. 如果总 loss 在降但关键子 loss 不降，优先检查权重和采样。
7. 后期根据收敛情况降低学习率，或切换到 L-BFGS。

重点在于：不要把所有训练问题都简单归因于学习率。学习率很重要，但它不能替代对模型公式、边界条件和损失设计本身的检查。

## 常见误区

- 误区一：学习率越大训练越快。  
  学习率过大往往会让训练直接震荡或发散。
- 误区二：学习率越小越稳定。  
  太小会让训练停滞，看起来像“没有学习”。
- 误区三：只要调学习率，就能修复所有训练问题。  
  如果 PDE、边界条件或自动微分有错，调学习率并不能根本解决。
- 误区四：总 loss 下降就代表学习率合适。  
  PINN 中必须同时检查各子 loss 和关键物理误差。
- 误区五：Adam 的学习率和 L-BFGS 的步长可以完全等同理解。  
  两者背后的更新机制不同，不能简单视为同一件事。

## 一页总结

- 学习率控制每次参数更新的步长。
- 梯度决定方向，学习率决定走多远。
- 学习率过小会让训练很慢，过大则会造成震荡、发散，甚至出现 `NaN`。
- 固定学习率简单直接，但未必适合训练全过程。
- 学习率调度和 warmup 可以让前期探索更快、后期收敛更稳。
- 学习率需要结合 batch size、优化器、损失权重和问题尺度一起考虑。
- 在 PINN 中，学习率还会影响 PDE 残差、边界条件和数据项之间的训练平衡，因此通常比普通监督学习更敏感。
